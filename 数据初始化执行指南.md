# 数据分析模块初始化执行指南

## 问题说明
数据分析模块的API已完整实现，但因数据库表为空导致前端显示无数据。

## 解决方案

### 方法1：执行SQL脚本（推荐）

```bash
# 进入SQL脚本目录
cd basketball-system/src/main/resources/sql

# 执行初始化脚本
mysql -u root -p basketball < init_analytics_data_fixed.sql
```

### 方法2：使用API手动触发

```bash
# 会员活跃度分析
curl -X POST "http://localhost:8080/api/admin/analytics/member/analyze?analysisDate=2025-01-22"

# 课程热度分析
curl -X POST "http://localhost:8080/api/admin/analytics/course/analyze?analysisDate=2025-01-22"

# 场地使用分析
curl -X POST "http://localhost:8080/api/admin/analytics/venue/analyze?analysisDate=2025-01-22"
```

## 验证结果

执行后访问以下页面验证：
- `/admin/analytics/member` - 会员活跃度分析
- `/admin/analytics/course` - 课程热度分析
- `/admin/analytics/venue` - 场地使用分析

## 注意事项

1. 确保MySQL服务已启动
2. 确保数据库`basketball`已存在
3. 确保有基础数据（用户、课程、场地）
4. 执行脚本前建议备份数据库

## 脚本说明

- `init_analytics_data.sql` - 原始版本（字段较少）
- `init_analytics_data_fixed.sql` - 完整版本（推荐使用）

完整版本包含：
- 会员活跃度的RFM分析字段
- 课程热度的转化率和评分字段
- 场地使用的详细统计字段
- 最近7天的历史数据
